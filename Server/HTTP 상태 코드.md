## TIL: HTTP 상태 코드 

## 개요

클라이언트(웹 브라우저, 모바일 앱 등)가 서버에 요청(Request)을 보내면, 서버는 요청에 대한 처리 결과와 함께 응답(Response)을 반환한다. 이때 서버가 응답의 상태를 3자리 숫자로 표현하여 알려주는 것을 **HTTP 상태 코드(HTTP Status Code)** 라고 한다. 이를 통해 클라이언트는 요청이 성공했는지, 실패했다면 어떤 이유로 실패했는지 직관적으로 파악할 수 있다.

-----

### 상태 코드의 5가지 범주

상태 코드는 첫 번째 숫자에 따라 5개의 클래스(Class)로 분류된다.

  * **1xx (Informational - 정보 제공):** 요청이 수신되었으며 처리가 계속 진행 중임을 의미한다. (실무에서는 거의 노출되지 않는다.)
  * **2xx (Successful - 성공):** 클라이언트의 요청이 성공적으로 수신, 이해 및 처리되었음을 의미한다.
  * **3xx (Redirection - 리다이렉션):** 요청을 완료하기 위해 클라이언트 측에서 추가적인 조치(주로 다른 URL로의 이동)가 필요함을 의미한다.
  * **4xx (Client Error - 클라이언트 오류):** 클라이언트의 요청에 잘못된 문법이 포함되어 있거나, 요청을 처리할 수 없음을 의미한다. (예: 잘못된 URL, 권한 부족)
  * **5xx (Server Error - 서버 오류):** 서버가 유효한 요청을 명백하게 수행하지 못했음을 의미한다. (예: 서버 다운, DB 연결 실패, 코드 버그)

-----

### 자주 사용되는 주요 상태 코드 상세

#### 2xx: 성공 (Success)

  * **`200 OK`**: 가장 통상적인 성공 코드이다. GET, POST, PUT, PATCH, DELETE 등 대부분의 요청이 정상적으로 처리되었을 때 반환한다.
  * **`201 Created`**: POST 요청 등을 통해 서버에 새로운 리소스가 성공적으로 생성되었을 때 반환한다.
  * **`204 No Content`**: 요청이 성공했으나, 클라이언트에게 반환할 데이터(Response Body)가 없음을 의미한다. 주로 DELETE 요청으로 데이터를 성공적으로 삭제했을 때 사용한다.

#### 3xx: 리다이렉션 (Redirection)

  * **`301 Moved Permanently`**: 요청한 리소스의 URI가 영구적으로 변경되었음을 알린다. 브라우저는 이 코드를 받으면 응답 헤더의 `Location`에 적힌 새로운 URL로 자동 이동한다.
  * **`304 Not Modified`**: 클라이언트가 캐시된 자원을 가지고 있으며, 서버의 자원이 수정되지 않았음을 알린다. 클라이언트는 새로 데이터를 다운로드하지 않고 저장된 캐시를 재사용하여 네트워크 대역폭을 절약한다.

#### 4xx: 클라이언트 오류 (Client Error)

  * **`400 Bad Request`**: 클라이언트가 잘못된 요청(문법 오류, 유효하지 않은 파라미터, 누락된 데이터 등)을 보내 서버가 처리할 수 없음을 의미한다.
  * **`401 Unauthorized`**: 해당 리소스에 접근하기 위한 인증(Authentication)이 필요함을 의미한다. 주로 로그인이 필요한 API에 비로그인 상태로 접근할 때 발생한다. (이름은 Unauthorized지만 실제로는 Unauthenticated의 의미에 가깝다.)
  * **`403 Forbidden`**: 인증(로그인)은 되었으나, 해당 리소스에 대한 접근 권한(Authorization)이 없음을 의미한다. (예: 일반 사용자가 관리자 전용 페이지에 접근 시도)
  * **`404 Not Found`**: 요청한 URL에 해당하는 리소스가 서버에 존재하지 않음을 의미한다.
  * **`405 Method Not Allowed`**: 요청한 HTTP 메서드(GET, POST 등)가 해당 리소스에서 지원되지 않음을 의미한다. (예: 읽기 전용 API에 POST 요청을 보낼 때)

#### 5xx: 서버 오류 (Server Error)

  * **`500 Internal Server Error`**: 서버 내부에서 알 수 없는 에러(예: NullPointerException 등 코드상 버그)가 발생하여 요청을 처리할 수 없음을 의미한다. 가장 포괄적인 서버 에러 코드이다.
  * **`502 Bad Gateway`**: 게이트웨이나 프록시 역할을 하는 서버(예: Nginx)가 그 뒷단의 서버(예: Spring Boot, Node.js)로부터 잘못된 응답을 받았음을 의미한다.
  * **`503 Service Unavailable`**: 서버가 오버로드되었거나 유지보수 중이어서 현재 요청을 처리할 수 없음을 의미한다.
