# 데코레이터 (Decorator)

## 데코레이터란?

데코레이터는 **기존 함수를 수정하지 않고, 그 함수에 새로운 기능을 덧붙이는 문법**이다. 이름 그대로 함수를 "장식(decorate)"한다.

`@` 기호를 함수 정의 위에 붙이는 것이 전부처럼 보이지만, 내부적으로는 **함수를 인자로 받아 새로운 함수를 반환하는 함수**가 동작하고 있다. FastAPI의 `@app.get("/")`, pytest의 `@pytest.fixture` 같은 것들이 전부 데코레이터다.

---

## 이해의 출발점 — 파이썬에서 함수는 객체다

데코레이터를 이해하려면 파이썬의 핵심 특성을 먼저 알아야 한다. 파이썬에서 함수는 **일급 객체(First-Class Object)** 다. 즉, 함수를 변수에 담거나, 다른 함수의 인자로 넘기거나, 반환값으로 돌려줄 수 있다.

```python
def greet():
    print("안녕!")

# 함수를 변수에 담기
say_hello = greet
say_hello()   # 안녕!

# 함수를 인자로 넘기기
def run(func):
    func()

run(greet)    # 안녕!
```

이것이 가능하기 때문에 데코레이터도 존재할 수 있다.


## 데코레이터의 동작 원리

### 1단계 — 함수를 감싸는 함수 (클로저)

```python
def my_decorator(func):       # 원본 함수를 인자로 받음
    def wrapper():            # 원본 함수를 감싸는 새 함수
        print("함수 실행 전")
        func()                # 원본 함수 호출
        print("함수 실행 후")
    return wrapper            # 새 함수를 반환
```

### 2단계 — 데코레이터 적용

```python
def say_hello():
    print("안녕!")

say_hello = my_decorator(say_hello)   # 수동으로 적용
say_hello()
```

```
함수 실행 전
안녕!
함수 실행 후
```

### 3단계 — @ 문법 (syntactic sugar)

`@` 는 위의 `say_hello = my_decorator(say_hello)` 를 짧게 쓰는 문법이다. 완전히 동일하게 동작한다.

```python
@my_decorator
def say_hello():
    print("안녕!")

say_hello()
```

즉, `@my_decorator`는 **"이 함수를 my_decorator에 넣고, 반환된 함수로 교체해라"** 는 뜻이다.

## 인자가 있는 함수에 데코레이터 적용

원본 함수가 인자를 받는다면 `wrapper`도 그 인자를 그대로 받아서 넘겨줘야 한다. `*args, **kwargs`를 쓰면 어떤 인자 조합이든 대응할 수 있다.

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):    # 모든 인자를 그대로 받음
        print(f"{func.__name__} 실행 전")
        result = func(*args, **kwargs)   # 원본 함수에 그대로 전달
        print(f"{func.__name__} 실행 후")
        return result                # 반환값도 그대로 전달
    return wrapper

@my_decorator
def add(a, b):
    return a + b

print(add(3, 5))
```

```
add 실행 전
add 실행 후
8
```

## functools.wraps — 함수 정체성 보존

데코레이터를 적용하면 원본 함수가 `wrapper`로 교체된다. 그러면 `__name__`, `__doc__` 같은 함수 메타 정보가 덮어써진다.

```python
@my_decorator
def say_hello():
    """인사하는 함수"""
    print("안녕!")

print(say_hello.__name__)   # wrapper  ← 원본 이름이 사라짐
print(say_hello.__doc__)    # None
```

`@functools.wraps(func)` 를 wrapper에 붙이면 원본의 메타 정보가 보존된다. **실무에서 데코레이터를 만들 때는 항상 붙이는 것이 관례**다.

```python
import functools

def my_decorator(func):
    @functools.wraps(func)    # 원본 함수의 이름, docstring 보존
    def wrapper(*args, **kwargs):
        print("실행 전")
        result = func(*args, **kwargs)
        print("실행 후")
        return result
    return wrapper

@my_decorator
def say_hello():
    """인사하는 함수"""
    print("안녕!")

print(say_hello.__name__)   # say_hello  ← 원본 이름 유지
print(say_hello.__doc__)    # 인사하는 함수
```

## 인자를 받는 데코레이터

데코레이터 자체에 인자를 넘기고 싶을 때는 **함수를 한 겹 더 감싼다**.

```python
@repeat(3)       # 데코레이터에 인자 전달
def say_hello():
    print("안녕!")
```

이 구조는 `say_hello = repeat(3)(say_hello)` 와 같다. `repeat(3)` 이 먼저 실행되어 데코레이터를 반환하고, 그 데코레이터가 `say_hello` 에 적용된다.

```python
import functools

def repeat(n):                       # 인자를 받는 가장 바깥 함수
    def decorator(func):             # 실제 데코레이터
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                func(*args, **kwargs)
        return wrapper
    return decorator                 # 데코레이터를 반환

@repeat(3)
def say_hello():
    print("안녕!")

say_hello()
```

```
안녕!
안녕!
안녕!
```

## 클래스로 만드는 데코레이터

`__call__` 메서드를 구현하면 클래스도 데코레이터로 쓸 수 있다. 상태(호출 횟수 등)를 저장해야 할 때 유용하다.

```python
import functools

class CountCalls:
    def __init__(self, func):
        functools.update_wrapper(self, func)
        self.func = func
        self.call_count = 0      # 상태를 인스턴스 변수로 유지

    def __call__(self, *args, **kwargs):
        self.call_count += 1
        print(f"{self.func.__name__} 호출 횟수: {self.call_count}")
        return self.func(*args, **kwargs)

@CountCalls
def say_hello():
    print("안녕!")

say_hello()   # say_hello 호출 횟수: 1
say_hello()   # say_hello 호출 횟수: 2
say_hello()   # say_hello 호출 횟수: 3
```

## 여러 데코레이터 중첩

데코레이터를 여러 개 쌓을 수 있다. **아래에서 위로** 적용된다.

```python
@decorator_A
@decorator_B
@decorator_C
def func():
    pass
```

이는 `func = decorator_A(decorator_B(decorator_C(func)))` 와 같다.

```python
def bold(func):
    @functools.wraps(func)
    def wrapper():
        return "<b>" + func() + "</b>"
    return wrapper

def italic(func):
    @functools.wraps(func)
    def wrapper():
        return "<i>" + func() + "</i>"
    return wrapper

@bold
@italic
def greet():
    return "안녕!"

print(greet())   # <b><i>안녕!</i></b>
```

## 실전 활용 패턴

### 실행 시간 측정

```python
import functools
import time

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} 실행 시간: {end - start:.4f}초")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)

slow_function()   # slow_function 실행 시간: 1.0012초
```

### 로그 남기기

```python
import functools
import logging

def log(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        logging.info(f"호출: {func.__name__}, 인자: {args}, {kwargs}")
        result = func(*args, **kwargs)
        logging.info(f"반환: {func.__name__} → {result}")
        return result
    return wrapper
```

### 권한 검사 (인증/인가)

```python
import functools

def login_required(func):
    @functools.wraps(func)
    def wrapper(user, *args, **kwargs):
        if not user.get("is_authenticated"):
            raise PermissionError("로그인이 필요합니다")
        return func(user, *args, **kwargs)
    return wrapper

def admin_required(func):
    @functools.wraps(func)
    def wrapper(user, *args, **kwargs):
        if not user.get("is_admin"):
            raise PermissionError("관리자 권한이 필요합니다")
        return func(user, *args, **kwargs)
    return wrapper

@login_required
@admin_required
def delete_user(user, user_id):
    print(f"{user_id}번 유저 삭제")
```

### 캐싱 — `@functools.lru_cache`

표준 라이브러리에도 캐싱 데코레이터가 내장되어 있다. 같은 인자로 호출하면 계산을 다시 하지 않고 저장된 결과를 돌려준다.

```python
import functools

@functools.lru_cache(maxsize=128)   # 최근 128개 결과 저장
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(50))   # 재귀가 수없이 중복되지만 순식간에 계산됨
```

## 프레임워크에서의 데코레이터

```python
# FastAPI — 라우터 등록
@app.get("/users/{user_id}")
def get_user(user_id: int):
    ...
# 내부 동작: get_user = app.get("/users/{user_id}")(get_user)
# app.get(...)이 데코레이터를 반환하고, 그 데코레이터가 get_user를 라우팅 테이블에 등록

# pytest — 픽스처 등록
@pytest.fixture
def db_session():
    ...
# pytest가 이 함수를 테스트 인프라에 등록함

# Python 내장 — 프로퍼티
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property             # getter
    def radius(self):
        return self._radius

    @radius.setter        # setter
    def radius(self, value):
        if value < 0:
            raise ValueError("반지름은 음수일 수 없습니다")
        self._radius = value
```
