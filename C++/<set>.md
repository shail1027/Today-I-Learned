## C++ `<set>` 헤더와 `std::set` 자료형


## 개요

C++ 표준 템플릿 라이브러리(STL)의 `<set>` 헤더에 포함된 `std::set`은 **고유한 원소들을 자동으로 정렬하여 저장**하는 연관 컨테이너(Associative Container)이다. 내부적으로는 균형 이진 탐색 트리(Balanced Binary Search Tree, 주로 레드-블랙 트리)로 구현되어 있어 데이터의 삽입, 삭제, 탐색에 있어 매우 효율적인 성능을 자랑한다.


## 주요 특징

1.  **중복 원소 허용 불가:** `set`에는 동일한 값을 가진 원소가 두 개 이상 존재할 수 없다. 이미 존재하는 값을 삽입(insert)하려고 시도하면 해당 요청은 무시된다. (중복을 제거할 때 매우 유용하다.)
2.  **자동 정렬:** 원소가 삽입될 때마다 내부적인 트리 구조에 의해 자동으로 오름차순 정렬된다. (내림차순 등 정렬 기준을 변경하고 싶다면 객체 생성 시 비교자(Comparator)를 전달할 수 있다.)
3.  **빠른 탐색 속도:** 트리 구조를 띠고 있기 때문에 특정 원소를 찾거나, 삽입, 삭제하는 연산의 시간 복잡도는 $O(\log n)$을 보장한다. 연속된 메모리 구조가 아니므로 인덱스(예: `s[0]`)를 통한 임의 접근(Random Access)은 불가능하며, 반복자(Iterator)를 사용해 접근해야 한다.


## 주요 멤버 함수 및 사용법

### 1. 선언 및 삽입 (`insert`)
`insert()` 함수를 사용하여 원소를 추가한다. 반환값으로 `pair<iterator, bool>`을 반환하는데, `bool` 값은 삽입 성공 여부(이미 있는 값이면 `false`)를 나타낸다.

```cpp
#include <iostream>
#include <set>

int main() {
    std::set<int> s;

    s.insert(5);
    s.insert(2);
    s.insert(8);
    s.insert(5); // 이미 5가 존재하므로 무시됨

    // 자동 정렬되어 2, 5, 8 순서로 출력됨
    for (int num : s) {
        std::cout << num << " ";
    }
    return 0;
}
```

### 2. 탐색 (`find`)
특정 값이 존재하는지 찾을 때 사용한다. 값을 찾으면 해당 원소를 가리키는 반복자를 반환하고, 찾지 못하면 `set::end()` 반복자를 반환한다.

```cpp
std::set<int> s = {1, 2, 3};

if (s.find(2) != s.end()) {
    std::cout << "2가 존재한다." << std::endl;
} else {
    std::cout << "2가 존재하지 않는다." << std::endl;
}
```

#### 3. 삭제 (`erase`)
특정 값이나 반복자가 가리키는 원소를 삭제한다.

```cpp
std::set<int> s = {10, 20, 30};

s.erase(20); // 값 20을 찾아 삭제한다.

// s.find()를 이용해 반복자를 얻어 삭제할 수도 있다.
auto it = s.find(30);
if (it != s.end()) {
    s.erase(it);
}
```

#### 4. 기타 유용한 함수
* `size()`: 현재 `set`에 포함된 원소의 개수를 반환한다.
* `empty()`: `set`이 비어있는지 여부를 반환한다 (`true` / `false`).
* `clear()`: `set`의 모든 원소를 삭제한다.

### 연관 컨테이너 비교 (`set` vs `multiset` vs `unordered_set`)

코딩 테스트나 실무에서는 상황에 맞게 알맞은 `set` 계열 컨테이너를 선택하는 것이 중요하다.

| 컨테이너 | 내부 구조 | 정렬 여부 | 중복 허용 | 시간 복잡도 (탐색/삽입/삭제) | 헤더 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **`std::set`** | 이진 탐색 트리 | O | **X** | $O(\log n)$ | `<set>` |
| **`std::multiset`** | 이진 탐색 트리 | O | **O** | $O(\log n)$ | `<set>` |
| **`std::unordered_set`**| 해시 테이블 | **X** | **X** | **평균 $O(1)$**, 최악 $O(n)$ | `<unordered_set>` |

* **`multiset`**: 중복된 원소를 저장해야 하면서 동시에 정렬이 필요할 때 사용한다.
* **`unordered_set`**: 원소의 정렬 상태가 중요하지 않고, 오직 데이터의 '존재 유무'를 가장 빠르게($O(1)$) 판단해야 할 때 사용한다.
