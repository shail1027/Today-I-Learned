## 컨테이너 순회: 반복자와 범위 기반 for 문

## 개요

C++에서 `vector`, `set`, `map` 등의 STL 컨테이너를 다룰 때, 내부의 데이터를 처음부터 끝까지 살펴보는 '순회(Traversal)'는 필수적인 작업이다. 이를 수행하는 근본적인 원리가 **반복자(Iterator)**이며, 이 반복자의 사용을  간결하게 만들어준 문법이 **범위 기반 for 문(Range-based for loop)**이다. 


## 1. 반복자(Iterator)란?

반복자는 쉽게 말해 **포인터(Pointer)를 객체 지향적으로 일반화한 것**이다. 

`vector`는 연속된 메모리 배열이고, `set`이나 `map`은 이진 트리 구조이며, `list`는 노드가 연결된 형태이다. 각 컨테이너마다 내부 구조가 완전히 다르기 때문에 원소에 접근하는 방식도 달라야 한다. 하지만 반복자를 사용하면 내부 구조가 무엇이든 간에 개발자는 **동일한 방법(++, --, * 등)으로 원소를 가리키고 순회**할 수 있게 된다.

---

### `begin()`과 `end()`의 의미 

컨테이너를 순회하기 위해서는 시작점과 끝점을 알아야 하며, 이를 위해 모든 STL 컨테이너는 `begin()`과 `end()` 멤버 함수를 제공한다.

* **`v.begin()`**: 컨테이너의 **첫 번째 원소**를 가리키는 반복자를 반환한다.
* **`v.end()`**: 컨테이너의 마지막 원소가 아니라, 마지막 원소의 **바로 다음(Past-the-end) 위치**를 가리키는 반복자를 반환한다. (실제 유효한 데이터가 있는 곳이 아니다.)

이러한 범위를 수학적으로는 **반개구간(Half-open range) `[begin, end)`** 이라고 부른다. 
이 방식을 사용하면 `begin() == end()`일 때 컨테이너가 비어있음을 매우 간단하게 확인할 수 있으며, 순회를 종료하는 조건(`it != v.end()`)을 명확하게 지정할 수 있다는 장점이 있다.


### 반복자를 이용한 순회 문법

전통적으로 반복자를 사용하여 컨테이너를 순회하는 방법은 다음과 같다.

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {10, 20, 30};

    // 1. 선언 방법 (자료형이 매우 길다)
    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        // *it를 통해 포인터처럼 역참조하여 실제 값에 접근한다.
        std::cout << *it << " "; 
        *it += 5; // 값 수정도 가능하다.
    }
    std::cout << "\n";

    // 2. auto 키워드를 활용한 단축 방법 (권장)
    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " "; 
    }

    return 0;
}
```

- **(`it++` vs `++it`):**
반복자를 증가시킬 때는 후위 증가 연산자(`it++`)보다 **전위 증가 연산자(`++it`)**를 사용하는 것이 좋다. 후위 연산자는 이전 상태를 임시 객체로 복사하여 보관해야 하므로, 구조가 복잡한 반복자에서는 불필요한 성능 저하를 유발할 수 있기 때문이다.

---

### 반복자의 다양한 종류

상황에 맞게 사용할 수 있도록 여러 형태의 반복자가 존재한다.

1.  **일반 반복자 (`iterator`)**: 
    * `begin()`, `end()`로 획득한다.
    * 읽기와 쓰기(수정)가 모두 가능하다.
2.  **상수 반복자 (`const_iterator`)**:
    * `cbegin()`, `cend()`로 획득한다. (C++11부터 추가)
    * 값을 읽을 수만 있고, 수정하는 것은 불가능하다. 데이터의 안전성을 보장해야 할 때 사용한다.
3.  **역방향 반복자 (`reverse_iterator`)**:
    * `rbegin()`, `rend()`로 획득한다.
    * 컨테이너를 뒤에서부터 앞으로 거꾸로 순회할 때 사용한다. `rbegin()`이 맨 뒤 원소를 가리키고, 여기서 `++` 연산을 수행하면 앞으로 이동한다.

```cpp
std::vector<int> v = {1, 2, 3};

// 역방향 순회 예시 (3 2 1 출력)
for (auto it = v.rbegin(); it != v.rend(); ++it) {
    std::cout << *it << " ";
}
```

## 2. 범위 기반 for 문 (Range-based for loop)   

C++11부터 도입된 `for (auto &i : v)` 형태의 문법이다. 이 문법은 놀라운 마법이 아니라, 내부적으로 위에서 설명한 **반복자(Iterator)를 사용하여 동작하도록 컴파일러가 코드를 자동 변환**해 주는 문법적 설탕(Syntactic Sugar)이다.

### `auto`와 `&`의 정확한 의미

#### 1\. `auto` (타입 추론)

`auto`는 변수를 선언할 때, 초기화되는 값을 보고 **컴파일러가 자료형을 자동으로 알아내어 지정**해 주는 키워드이다.
`std::vector<pair<int, string>>`과 같이 자료형이 매우 길고 복잡할 때 타이핑을 줄여주며, 나중에 컨테이너의 자료형이 바뀌더라도 `for` 문을 수정할 필요가 없어 유지보수에 유리하다.

#### 2\. `&` (참조, Reference)

`&`가 없다면 컨테이너의 원소를 순회할 때마다 \*\*'값의 복사(Copy)'\*\*가 일어난다. 만약 원소가 단순한 `int`라면 복사 비용이 적지만, 원소가 긴 `string`이거나 복잡한 `struct`/`class`라면 매번 복사하는 데 엄청난 메모리와 시간이 낭비된다.
`&`를 붙이면 원본 메모리 주소를 직접 **참조**하므로, 복사 비용이 전혀 들지 않으며 순회 중 원본 값을 수정할 수도 있다.


### 형태별 비교 

상황에 따라 아래 세 가지 형태 중 하나를 선택해서 사용해야 한다.

| 형태 | 예시 | 복사 발생 | 원본 수정 | 설명 및 사용처 |
| :--- | :--- | :---: | :---: | :--- |
| **Value** | `for (auto i : v)` | **O** | **X** (복사본 수정됨) | 원소의 크기가 작고(`int`, `char`), 원본을 절대 수정하지 않으며, 순회 중 값을 편하게 변경해보고 싶을 때 사용. |
| **Reference** | `for (auto &i : v)` | **X** | **O** | 순회하면서 원본 컨테이너의 **값을 직접 변경**해야 할 때 사용. |
| **Const Ref** | `for (const auto &i : v)` | **X** | **X** | **가장 권장되는 형태.** 원본을 수정하지 않고 단순히 값을 '읽기'만 할 때, 복사 비용 없이 가장 안전하고 빠르게 순회할 수 있다. |

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3};

    // 1. 값 복사 (원본 안 바뀜)
    for (auto i : v) { i *= 10; }
    // v는 여전히 {1, 2, 3}

    // 2. 참조 (원본 바뀜)
    for (auto &i : v) { i *= 10; }
    // v는 이제 {10, 20, 30}

    return 0;
}
```


### 순회 가능한 자료형 (Iterable Types)

범위 기반 `for` 문을 사용하려면 해당 자료형이 내부적으로 \*\*반복자(Iterator)\*\*인 `begin()`과 `end()` 함수를 지원해야 한다.

1.  **STL 컨테이너:** `vector`, `list`, `deque`, `set`, `map`, `unordered_map` 등 대부분의 C++ 표준 컨테이너는 모두 지원한다.
2.  **`std::string`:** 문자열도 문자의 배열이므로 한 글자씩(`char`) 순회할 수 있다.
3.  **C 스타일 배열:** 크기가 고정된 배열(`int arr[5];`)도 순회가 가능하다.
      * **주의사항:** 배열이 함수의 매개변수로 전달되어 포인터로 붕괴(Decay)된 경우(`void func(int arr[])`)에는 배열의 전체 크기를 알 수 없으므로 범위 기반 for 문을 **사용할 수 없다.**


### Map을 순회할 때의 팁 (C++17 구조적 바인딩)

`std::map`이나 `unordered_map`을 `auto`로 순회하면 원소들이 `std::pair` 형태로 튀어나오기 때문에 `i.first`, `i.second`로 접근해야 해서 코드가 다소 지저분해진다.

C++17부터 도입된 \*\*구조적 바인딩(Structured Binding)\*\*을 사용하면 이를 매우 깔끔하게 풀 수 있다.

```cpp
#include <iostream>
#include <map>

int main() {
    std::map<std::string, int> m = {{"Apple", 1}, {"Banana", 2}};

    // 기존 방식 (C++11)
    for (const auto &item : m) {
        std::cout << item.first << ": " << item.second << "\n";
    }

    // 구조적 바인딩 (C++17)
    for (const auto &[key, value] : m) {
        std::cout << key << ": " << value << "\n";
    }

    return 0;
}
```


## 3. 두 방식의 비교 및 실전 적용 가이드

그렇다면 언제 무엇을 써야 할까? 두 방식의 장단점을 이해하고 상황에 맞게 적절히 골라 써야 한다.

| 비교 항목 | 범위 기반 for 문 (`for (auto &i : v)`) | 반복자 순회 (`for (auto it = v.begin();...`) |
| :--- | :--- | :--- |
| **장점** | 코드가 압도적으로 간결하고 직관적이며 실수할 여지가 적다. | 인덱스 제어, 역방향 순회 등 세밀하고 유연한 조작이 가능하다. |
| **순회 범위** | **무조건** 처음부터 끝까지 전체를 순회한다. | 시작점과 끝점을 임의로 설정하여 **부분 순회**가 가능하다. |
| **원소 삭제/삽입**| 순회 도중 `erase`나 `insert`를 사용하면 반복자 무효화(Iterator Invalidation)가 발생해 런타임 에러가 날 확률이 높다. | `it = v.erase(it);` 형태로 안전하게 현재 원소를 지우며 다음으로 넘어갈 수 있다. |
| **실전 권장** | 단순 탐색, 출력, 전체 값 갱신 등 **90% 이상의 일반적인 상황**에서 최우선으로 사용한다. | 순회 중 원소를 지워야 하거나, `<algorithm>` 헤더의 함수(`sort`, `unique` 등)에 범위를 지정해 줄 때 필수적으로 사용한다. |
