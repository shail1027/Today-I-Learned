## React & Vite Basics

## 1. React란 무엇인가? 

**React(리액트)** 는 사용자 인터페이스(UI)를 만들기 위한 **자바스크립트 라이브러리**다.

* **역할:** 화면에 무엇이 보여야 하는지, 버튼을 누르면 어떤 일이 일어나야 하는지(로직)를 담당한다.
* **특징:** 컴포넌트(Component) 기반으로 UI를 레고 블록처럼 조립해서 만든다.

하지만 React만으로는 브라우저에서 바로 실행되기 어렵다. 브라우저는 React의 문법(JSX)을 바로 이해하지 못하기 때문이다. 

## 2. Vite란 무엇인가?

**Vite**는 프론트엔드 개발 경험을 향상시켜주는 **빌드 도구(Build Tool)** 다. 프랑스어로 '빠르다'는 뜻을 가지고 있다.

* **역할:** React가 작성한 코드를 브라우저가 이해할 수 있는 자바스크립트로 변환(Transpile)하고, 개발 서버를 띄워주고, 배포할 때 파일을 예쁘게 묶어준다(Bundling).
* **특징:** 기존의 빌드 도구(Webpack 등)보다 압도적으로 빠르다. 브라우저의 기본 기능(ES Modules)을 활용하기 때문이다.

## 3. 둘의 관계: 왜 같이 쓰는가?

React와 Vite는 **상호 보완 관계**다.

### 3-1. 브라우저의 한계
우리가 React로 코드를 짤 때는 `.jsx`나 `.tsx` 파일을 쓴다.
```jsx
// React 코드 (App.jsx)
function App() {
  return <h1>Hello Vite!</h1>;
}
```

브라우저는 이 코드를 읽지 못해 에러를 뱉는다. 이따, **Vite**는 이 코드를 낚아채서 일반 자바 스크립트(`.js`)로 번역하여 브라우저에 띄워준다.

### 3-2. HMR

과거(Webpack 시절)에는 소스 한 줄을 수정하고 저장하면, 전체를 다시 읽느라 3~5초씩 걸렸다. 하지만 Vite는 **HMR(Hot Module Replacement)** 기능이 매우 강력해서, 저장하자마자 화면이 바뀐다.
